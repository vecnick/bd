# Ответ на вопрос 4

Дано:

```
u = {(a), (b), (c)},  v = {(b), (c), (d)}

t(X,Y) :- u(X), u(Y), not(v(X)).
r(X)   :- u(X), v(Y), not(t(X,Y)).
s(X)   :- r(Y), t(Y,X), not(r(X)).
```

Экстенсиональные предикаты: `u/1`, `v/1`. Интенсивиональные: `t/2`, `r/1`, `s/1`.

## 4(a) safe? stratified?

Программа **safe**, потому что во всех правилах переменные из отрицательного литерала попадают в положительные атомы тела:

- `t`: отрицательный `not(v(X))` использует `X`, а `X` уже ограничен положительными `u(X), u(Y)`.
- `r`: отрицательный `not(t(X,Y))` использует `X` и `Y`, которые присутствуют в положительных `u(X)` и `v(Y)`.
- `s`: отрицательный `not(r(X))` использует `X`, а `X` присутствует в положительном атоме `t(Y,X)`.

Программа **stratified**. Негативные зависимости между интенсивиональными предикатами:

- `r` использует `not(t(X,Y))` => `t` ниже `r`.
- `s` использует `not(r(X))` => `r` ниже `s`.

Следовательно, можно взять такие стратумы:

- **Страта 0:** `t`
- **Страта 1:** `r`
- **Страта 2:** `s`

## 4(b) Интенсивиональные отношения в стратифицированной семантике

### Шаг 1 (stratum 0): вычисляем `t`

Правило:
` t(X,Y) :- u(X), u(Y), not(v(X)).`

`not(v(X))` истинно только когда `X = a` (так как `v` содержит `b,c,d`, а `a` в `v` отсутствует).
Поэтому:

- для `X=a` и любого `Y` из `u={a,b,c}` получаем `t(a,a)`, `t(a,b)`, `t(a,c)`
- для `X=b,c` `not(v(X))` ложно => `t(b,*)` и `t(c,*)` не выводятся

Итого:
`t = {(a,a), (a,b), (a,c)}`

### Шаг 2 (stratum 1): вычисляем `r`

Правило:
`r(X) :- u(X), v(Y), not(t(X,Y)).`

Возможные `X` из `u`: `a,b,c`. Проверяем `not(t(X,Y))` для `Y` из `v={b,c,d}`:

- `r(a)` нужно, потому что `t(a,d)` отсутствует (при `Y=d`).
- `r(b)` нужно, потому что `t(b,b)` отсутствует.
- `r(c)` нужно, потому что `t(c,c)` отсутствует.

Итого:
`r = {(a), (b), (c)}`

### Шаг 3 (stratum 2): вычисляем `s`

Правило:
`s(X) :- r(Y), t(Y,X), not(r(X)).`

Здесь `t(Y,X)` возможно только когда `Y=a` (так как `t` имеет кортежи только с первым аргументом `a`).
Но `r(a)` истинно, а значит для `X=a` литерал `not(r(X))` ложно; для `X=b,c` также `not(r(X))` ложно.

Итого:
`s = ∅`

**Ответ (stratified model):**
- `t = {(a,a), (a,b), (a,c)}`
- `r = {(a), (b), (c)}`
- `s = ∅`

## 4(c) Другой минимальный model

Одна альтернатива:

```
t' = {(a,a), (a,b), (a,c), (a,d)}
r' = {(b), (c)}
s' = ∅
```

### Почему это модель

- Правило для `t` **разрешает** добавлять `t(a,d)`: оно не обязано быть истинным правилом `t` (его там нет, потому что `d` не входит в `u` как второй аргумент), но и не нарушает другие правила.
- `r(a)` в этой модели не выводится, потому что теперь `t(a,Y)` истинно для всех `Y` из `v={b,c,d}`, и следовательно `not(t(a,Y))` нигде не срабатывает.
- `r(b)` и `r(c)` при этом по-прежнему необходимы, потому что `t(b,b)` и `t(c,c)` отсутствуют (а в правиле `r` есть `v(b)` / `v(c)`).
- Для `s` необходимо `not(r(X))`. Так как `r(b)` и `r(c)` истинны, а `r(a)` ложен, но `t(a,X)` отсутствует для `X` кроме `a` (и при `X=a` условие `not(r(a))` как раз истинно, однако нужные `t(Y,a)` не возникают с `Y` имеющим `r(Y)`), получаем `s' = ∅`.

### Почему это минимальная модель

Если убрать любой элемент из `r'` или из обязательной части `t'`:

- исчезновение `r(b)` или `r(c)` сделает нарушенным правило `r`, так как для соответствующих `X` останется хотя бы один `Y` из `v`, для которого `t(X,Y)` отсутствует;
- исчезновение обязательных `t(a,a)`, `t(a,b)`, `t(a,c)` нарушит правило `t`.

`t(a,d)` при этом минимален в том смысле, что если его удалить, снова появится необходимость выводить `r(a)`.
