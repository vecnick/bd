# Ответ на вопрос 3

Дано:

```
bi(X,Y) :- g(X,Y).
bi(Y,X) :- g(X,Y).

even(X,Y) :- bi(X,Z), bi(Z,Y).
even(X,Y) :- bi(X,U), bi(U,V), even(V,Y).

g = {(a, b), (b, c), (c, d)}.
```

Нужно:
1) найти минимальные модели базовой программы;
2) добавить правило `onlyodd(X,Y) :- not(even(X,Y))` и выяснить безопасность;
3) сделать программу безопасной;
4) найти минимальные модели после правки и указать, какая соответствует стратифицированной семантике.

## 1) Минимальная модель базовой

Так как в программе нет отрицаний, минимальная модель единственная.

### `bi`
`bi` — это ребра `g` и их обратные:
- `(a,b)`, `(b,c)`, `(c,d)`
- `(b,a)`, `(c,b)`, `(d,c)`

Итого:
`bi = {(a,b),(b,c),(c,d),(b,a),(c,b),(d,c)}`

### `even`
`even(X,Y)` получается из путей длины 2 и более чётной длины в графе `bi`.
В этом линейном графе чётные пути существуют тогда и только тогда, когда `X` и `Y` лежат в одной части:
- `{a,c}` и `{b,d}`

Итого:
`even = {(a,a),(a,c),(c,a),(c,c),(b,b),(b,d),(d,b),(d,d)}`

## 2) Добавляем `onlyodd(X,Y) :- not(even(X,Y))`

### Безопасность
Нет: правило небезопасно, потому что переменные `X` и `Y` встречаются **только** в отрицательном литерале `not(even(X,Y))` и не ограничиваются никакими положительными атомами в теле правила.

### Как сделать безопасным (пример правки)
Один из способов — ввести домен через имеющиеся факты `g` (или `bi`) и привязать `X` и `Y` положительными атомами:

```prolog
dom(X) :- g(X,_).
dom(X) :- g(_,X).

onlyodd(X,Y) :- dom(X), dom(Y), not(even(X,Y)).
```

После этого `X` и `Y` ограничены доменом, и правило становится `safe`.

## 3) Минимальные модели после безопасной правки

Позитивные правила по `bi` и `even` по-прежнему принуждают (обязательны) факты:
`even0 = {(a,a),(a,c),(c,a),(c,c),(b,b),(b,d),(d,b),(d,d)}`

Оставшиеся пары в домене (чётность расстояния между долями разная) — это:
`cross = {(a,b),(a,d),(c,b),(c,d),(b,a),(b,c),(d,a),(d,c)}`

После добавления `onlyodd` на этих `cross` парах остаётся свобода для минимальных моделей:
- для каждой пары из `cross` можно либо добавить `even(X,Y)` (и тогда `onlyodd(X,Y)` не требуется),
- либо оставить `even(X,Y)` не выводимым и тогда `onlyodd(X,Y)` должен быть истинным по правилу с `not(even(X,Y))`.

То есть минимальные модели задаются выбором множества `S ⊆ cross`:

`even = even0 ∪ S`  
`onlyodd = cross \ S`

Таких минимальных моделей `2^8` (потому что `|cross| = 8`).

## 4) Какая из минимальных моделей соответствует стратифицированной семантике

В стратифицированной семантике сначала фиксируется наименьшая (positive) часть:
`even = even0`,
а затем `onlyodd` — точный комплемент `even` по домену.

Значит, нужная минимальная модель — это случай:
`S = empty`

то есть:
`even = even0`  
`onlyodd = cross = {(a,b),(a,d),(c,b),(c,d),(b,a),(b,c),(d,a),(d,c)}`
