# Ответ на вопрос 2

## Критерий `safe`

Правило считается `safe`, если все переменные из отрицательных литералов (`not(...)`) встречаются хотя бы в одном положительном атоме тела этого же правила.

## program 1

`safe`: да.

В правиле:
`not(X=Y)` использует переменные `X` и `Y`, и при этом `X` и `Y` присутствуют в положительных `parent(Z,X)` и `parent(Z,Y)`, значит переменные ограничены доменом.

`stratified`: да.

Негативных зависимостей между интенсивиональными предикатами нет, значит достаточно одного стратума.

Страта 0: `sister`, `aunt`.

## program 2

`safe`: нет (unsafe).

В правиле:
`root(X) :- not(parent(Y,X)).`

переменная `Y` присутствует только в отрицательном литерале `not(parent(Y,X))` и нигде больше в теле правила не ограничивается положительным атомом.

`stratified`: да.

Отрицание направлено только на `parent`, который является экстенсиональным предикатом, а рекурсия в `same_generation` использует только положительные вхождения.

Страта 0: `root`.
Страта 1: `same_generation`.

## program 3

`safe`: да.

Во всех правилах переменные из `not(friend(...))`, `not(enemy(...))` и `not(connects_new_enemies(...))` присутствуют в положительных (`friend(...)`, `enemy(...)`, `person(...)`, `connected_after(...)`).

`stratified`: да.

Негативные зависимости по предикатам:
- `enemy` использует `not(friend(...))` => `friend` ниже `enemy`.
- `connects_new_enemies` использует `not(friend(...))` => `friend` ниже `connects_new_enemies`.
- `potential_friend` использует `not(enemy(...))` и `not(connects_new_enemies(...))` => `enemy` и `connects_new_enemies` ниже `potential_friend`.

Соответственно:
Страта 0: `friend`, `connected_after`.
Страта 1: `enemy`, `connects_new_enemies`.
Страта 2: `potential_friend`.
