# Вопрос 4 (Session 6: OLAP) — tightness greedy bound

Нужно показать tightness для границы:

`Bgreedy / BOpt >= 1 - ((k-1)/k)^k = 1 - (1 - 1/k)^k`

для view selection в lattice.

## Идея: сводим к классической задаче maximum k-coverage

Пусть benefit множества выбранных view задан как суммарный вес “покрытых” элементов (как в maximum k-coverage). Тогда greedy-алгоритм для view selection совпадает по анализу с greedy для k-stage covering.

Для maximum k-coverage существует семейство instances, где greedy даёт ровно ak times оптимума, а:

`ak = 1 - (1 - 1/k)^k`.

## Конструкция instances (для произвольного k)

Рассмотрим универсум элементов как матрицу:

`U = { (i,j) | 0 <= i <= k, 1 <= j <= k }`

Строки i=0..k и столбцы j=1..k. Для каждого i зададим специальные веса ui,j так, чтобы выполнялись два свойства (это стандартная конструкция Dasgupta–Janardan–Sherwani, изложенная в [Hochbaum & Pathria, *Maximum k-Coverage*]):

1) greedy выбирает подмножества в порядке:
`[Rk, R(k-1), ..., R1]`,
где семейство допустимых subset’ов `R` устроено так, что каждый subset состоит из элементов либо одной колонки, либо одной строки (с исключением строки 0 для выбора).

2) суммарный вес элементов, которые попали в строки 1..k, равен:

`a_k * (вес всего U)`,

а оптимальный выбор k подмножеств (например, все k столбцов) покрывает весь U.

Тогда:

`Bgreedy = a_k * BOpt`

и, значит,

`Bgreedy/BOpt = 1 - ((k-1)/k)^k`.

Тем самым граница greedy для произвольного k является tight (в худшем случае).
