# Вопрос 3 (Session 6: OLAP) — размеры view, greedy и benefit

Дано: куб с измерениями `Product` и `Date`, мера `TotalSales`.

Иерархия по `Date`:
- `year`
- `month`
- `week`
- `Date` (день)

Нет иерархии по `Product` (но в кубе всё равно есть уровень “ALL products”).

100 продуктов.
3 года данных: `1095` дней, `157` недель, `36` месяцев.

## 3(a) Размеры разных view (dense)

Рассмотрим view как сочетание:
- либо конкретные продукты (100),
- либо агрегирование по всем продуктам (1),
умноженное на выбранный уровень иерархии Date:
- day (1095),
- week (157),
- month (36),
- year (3),
- all-dates (1).

Тогда размеры:

- `(Product, day)`   : `100 * 1095 = 109500`
- `(Product, week)`  : `100 * 157  = 15700`
- `(Product, month)` : `100 * 36   = 3600`
- `(Product, year)`  : `100 * 3    = 300`
- `(Product, ALL)`   : `100 * 1    = 100`

- `(ALL products, day)`   : `1 * 1095 = 1095`
- `(ALL products, week)`  : `1 * 157  = 157`
- `(ALL products, month)` : `1 * 36   = 36`
- `(ALL products, year)`  : `1 * 3    = 3`
- `(ALL products, ALL)`   : `1 * 1     = 1`

## 3(b) Жадный алгоритм: выбрать 2 view

В классической постановке view selection greedy-выбирает view с максимальным **добавочным** benefit при заданной текущей обстановке, где одна materialized view позволяет “закрывать” запросы на более агрегированных уровнях.

Самая детальная view `(Product, day)` позволяет получить все остальные view только агрегацией по Date и/или по Product, поэтому:
- на первом шаге greedy выбирает `(Product, day)`;
- на втором шаге дополнительный benefit равен 0 (все остальные view уже покрыты).

Если нужен конкретный выбор второго view при нулевой добавке, можно взять следующий по размеру, например `(Product, week)`.

## 3(c) Total benefit для этих двух view

Поскольку `(Product, day)` покрывает все остальные view, total benefit для набора из 2 view равен сумме benefit всех view, которые покрываются.

При допущении, что benefit каждого view пропорционален её размеру (как в типовых задачах по примеру lattice view selection):

`TotalBenefit = 109500 + 15700 + 3600 + 300 + 100 + 1095 + 157 + 36 + 3 + 1 = 130492`
